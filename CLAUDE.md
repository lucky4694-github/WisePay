# WisePay プロジェクト ルール

## ファイル修正時の必須ルール

### 修正ログの記録

**どのファイルを修正しても**、以下を必ず行うこと：

1. **修正したファイル自体**の先頭付近（コメントアウト可能な箇所）に修正履歴を追記する
2. **index.html**（修正の有無に関わらず）に同じ内容を必ず記録する

### 記録フォーマット

```
修正: YYYY-MM-DD HH:MM — [修正内容の簡潔な説明]
```

例：
```
修正: 2026-05-22 19:50 — freee 급여 CSV 업로드 UI 추가 + GAS importPayrolls 핸들러
```

### index.html への記録場所

`index.html` の `<head>` 内、最上部のコメントブロックに追記する：

```html
<!--
  修正: 2026-05-23 10:00 — app.js: 급여 계산 로직 수정
  修正: 2026-05-22 19:50 — freee CSV 업로드 UI 추가 + GAS importPayrolls 핸들러
-->
```

新しい修正は**上に追加**（最新が一番上）。

### 厳守事項

- ファイルを1行でも変更したら、必ずこのルールを適用する
- index.html は修正していなくても**必ず更新**する
- 日時は実際の修正時刻を記載（不明な場合はユーザーに確認する）
- 内容は日本語・韓国語どちらでも可
